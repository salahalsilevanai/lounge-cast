# Lounge Cast

#### Video Demo: https://www.youtube.com/watch?v=sk0YoY14DMM

#### Description:

**Watch & Listen Together** — a Chrome extension that lets you and your friends watch OSN+ or YouTube videos in perfect sync, with a built-in chat, from anywhere.

Lounge Cast pairs a lightweight Chrome extension (Manifest V3) with a small Socket.IO relay server to keep playback state — play, pause, seek, and even URL/episode changes — synchronized across everyone in a room, alongside a real-time text chat overlay.

---

## Features

- 🎬 **Synced playback** — play, pause, and seek events are broadcast to everyone in the room in real time
- 📺 **Episode/URL sync** — when the host changes videos or moves to the next episode, everyone follows automatically
- 💬 **In-page chat overlay** — a slide-out chat panel injected directly into OSN+/YouTube pages
- 🏠 **Room-based sessions** — create or join a room with a shareable Room ID
- 👤 **Editable display names** — random usernames are generated by default, but can be changed at any time
- ⌨️ **Playback hotkeys** — `p` pause, `s` play, `r` restart, `j`/`l` seek backward/forward 10s
- 🟢 **Live status indicator** — popup UI shows whether the extension is currently connected

## Architecture

```
┌─────────────────────┐        Socket.IO         ┌──────────────────┐
│   Chrome Extension   │ ◄──────(WebSocket)──────►│   Node.js Server  │
│                       │                          │                  │
│  ┌─────────────────┐  │                          │  socket.io room  │
│  │  popup.js/html   │  │                          │  state tracking  │
│  └────────┬─────────┘  │                          │  (per Room ID)   │
│           │ chrome.tabs │                          └──────────────────┘
│           ▼            │
│  ┌─────────────────┐  │
│  │   Content.js     │  │
│  │ (injected script)│  │
│  └────────┬─────────┘  │
│           │ chrome.runtime.sendMessage
│           ▼            │
│  ┌─────────────────┐  │
│  │  background.js   │  │
│  │ (service worker)  │  │
│  └─────────────────┘  │
└─────────────────────┘
```

- **`background.js`** — the MV3 service worker. Owns the persistent Socket.IO connection to the relay server, tracks which browser tab has joined which room (`tabRooms`), and routes inbound events only to the tab(s) actually in that room.
- **`Content.js`** — injected into `osnplus.com` and `youtube.com` pages. Locates the `<video>` element, listens for local play/pause/seek events, renders the chat overlay UI, and applies inbound sync events from peers.
- **`popup/`** — the toolbar popup UI for creating/joining/leaving a room and changing your display name.
- **`server.js`** — a minimal Socket.IO server that tracks per-room playback state (`isPlaying`, `currentTime`, `url`) as the single source of truth, and rebroadcasts events to everyone else in a room.

## Getting Started

### 1. Server

The relay server tracks room state and rebroadcasts sync events. It depends on `socket.io` (with `nodemon` for local dev).

```bash
cd server
npm install
node server.js
```

For development with auto-restart on file changes:

```bash
npm run devStart
```

By default it listens on port `3000`. Update the connection URL in `client/background.js` to point at wherever you deploy it:

```javascript
const socket = io("https://your-server-domain.com/", {
  autoConnect: true,
  transports: ["websocket"],
});
```

### 2. Extension

1. Open `chrome://extensions`
2. Enable **Developer mode**
3. Click **Load unpacked** and select the `client/` folder
4. Pin the Lounge Cast icon to your toolbar

### 3. Usage

1. Open OSN+ or YouTube and start playing something
2. Click the Lounge Cast icon
3. **Create Room** to start a new session, or enter a **Room ID** and **Join Room** to join a friend
4. Share the Room ID with whoever you're watching with
5. Use the in-page **Chat** button to open the chat panel, or the hotkeys (`p` / `s` / `r` / `j` / `l`) to control playback

## Tech Stack

- **Extension:** Manifest V3, vanilla JavaScript, Chrome Extension APIs (`storage`, `tabs`, `runtime`)
- **Real-time transport:** [Socket.IO](https://socket.io/) (WebSocket-only transport)
- **Server:** Node.js + `socket.io` (`nodemon` for local development)

## Permissions

| Permission  | Why it's needed                                          |
| ----------- | -------------------------------------------------------- |
| `activeTab` | Send messages to the currently active tab from the popup |
| `tabs`      | Query the active tab and relay events to specific tabs   |
| `storage`   | Persist username, room ID, and connection status locally |

## Roadmap / Known Limitations

- Currently supports **one room per tab** — a tab can't spectate multiple rooms at once
- Only tested against `osnplus.com` and `youtube.com`; other video sites aren't matched by the content script yet
- No authentication — anyone with a Room ID can join
- Chat history is not persisted between sessions

## Acknowledgments

Lounge Cast was designed and built by me from the ground up. Towards the end of the project, I used Claude (Anthropic) as a debugging and code-review assistant to help track down and fix a handful of specific issues we worked through together:

- A Manifest V3 service worker lifecycle bug, where Chrome terminating the idle background script wiped its in-memory room-tracking state and silently desynced peers until a tab was reloaded — fixed with a `rebuildTabRooms()` recovery routine that runs on every socket reconnect.
- A race condition where a peer-triggered URL/episode change could echo back out to the server as if it were a local navigation.
- Missing server-side handling for peers who disconnected without explicitly leaving a room, which left stale peers in the roster and leaked room state over time.

All core architecture, features, and implementation are my own; Claude's role was reviewing existing code, identifying the root cause of specific bugs, and helping implement the corresponding fixes.
